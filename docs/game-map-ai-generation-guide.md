# 游戏地图AI生成功能使用指南

## 概述

游戏地图AI生成功能是一个强大的工具，允许用户先用简单的图块绘制地图轮廓，然后使用AI技术生成专业级的游戏地图。该功能将传统的tilemap编辑器与现代AI生成技术相结合，为游戏开发者提供高效的地图创作体验。

## 功能特点

1. **保留传统tilemap编辑功能** - 所有现有的图块放置、撤销/重做、导出功能保持不变
2. **AI增强生成** - 基于像素轮廓生成高质量游戏地图
3. **参数化控制** - 支持多种风格、尺寸、游戏类型的定制
4. **实时预览** - 草图预览和生成结果即时显示
5. **历史记录** - 自动保存生成历史，方便回溯和重用

## 使用流程

### 第一步：访问游戏地图编辑页面

1. 访问 `http://localhost:8083/create-game/asset`
2. 点击"游戏地图"卡片进入编辑页面

### 第二步：绘制地图草图

1. **选择图块**：从左侧图块面板中选择合适的图块类型
2. **绘制轮廓**：在画布上点击放置图块，绘制地图的基本轮廓
   - 建议使用不同颜色的图块表示不同的地形区域
   - 可以使用障碍物图块标记不可通行区域
3. **调整网格**：根据需要调整网格大小（16×16、32×32、64×64）
4. **草图预览**：右侧面板会实时显示草图预览和图块数量统计

### 第三步：设置AI生成参数

在右侧AI参数面板中设置以下参数：

#### 1. 风格设置
- **地图风格**：选择地图的艺术风格
  - 奇幻风格：适合魔幻、奇幻类游戏
  - 科幻风格：适合未来、科技类游戏
  - 中世纪风格：适合历史、骑士类游戏
  - 现代风格：适合都市、现代类游戏
  - 赛博朋克：适合高科技低生活风格
  - 蒸汽朋克：适合维多利亚时代蒸汽科技风格

#### 2. 地图尺寸
- **小型 (512×512)**：适合小场景、室内地图
- **中型 (1024×1024)**：标准尺寸，适合大多数游戏场景
- **大型 (2048×2048)**：适合大型开放世界区域
- **自定义尺寸**：根据特定需求定制

#### 3. 游戏设置
- **游戏类型**：选择地图适用的游戏类型
  - 角色扮演 (RPG)：适合剧情驱动的地图
  - 策略/战棋：适合战术布局的地图
  - 平台跳跃：适合平台游戏的地图
  - 沙盒游戏：适合自由探索的地图
  - 冒险游戏：适合解谜探索的地图
  - 解谜游戏：适合谜题设计的地图

#### 4. 微调模型
- **基础模型**：快速生成，适合原型设计
- **专业模型**：高质量生成，适合正式项目
- **定制模型**：需要特定授权的专业模型

#### 5. 参考图片（可选）
- 上传参考图片，AI会参考图片的风格和元素
- 支持JPG、PNG格式，最大5MB

### 第四步：生成地图

1. 点击"🚀 生成游戏地图"按钮
2. 系统会显示生成进度和状态
3. 生成完成后，结果会显示在右侧面板

### 第五步：使用生成结果

生成完成后，您可以：

1. **预览地图**：查看AI生成的地图效果
2. **下载地图**：点击"⬇️ 下载地图"按钮保存PNG文件
3. **重新生成**：点击"🔄 重新生成"使用相同参数重新生成
4. **全屏查看**：点击生成的地图图片进入全屏模式
5. **保存到素材库**：点击"💾 保存"按钮将地图保存到个人素材库

## 高级功能

### 草图数据转换

系统会自动将您的tilemap草图转换为AI可理解的轮廓数据：
- 提取图块位置和类型
- 检测轮廓边界
- 简化轮廓数据
- 转换为AI输入格式

### 面板布局调整

- 拖动中间的分隔符可以调整左右面板的大小
- 面板大小会自动保存，下次访问时保持相同布局

### 键盘快捷键

- **Ctrl/Cmd + Enter**：快速生成地图
- **Esc**：取消正在进行的生成任务
- **Ctrl/Cmd + S**：保存地图到素材库

### 历史记录

- 所有生成的地图都会自动保存到历史记录
- 最多保存50条历史记录
- 可以查看历史生成的地图和参数

## 技术架构

### 前端组件

1. **TilemapEditor** - 传统的图块编辑器，已增强支持草图数据导出
2. **AIParamsPanel** - AI参数输入面板，处理参数验证和保存
3. **SketchConverter** - 草图数据转换器，将tilemap转换为AI轮廓
4. **GameMapAPIClient** - API客户端，与后端通信
5. **GameMapGeneratorApp** - 主应用程序，集成所有组件
6. **ResultPreview** - 结果预览组件，显示和管理生成结果

### 后端API

1. **POST /api/game-map/generate** - 提交地图生成请求
2. **GET /api/game-map/status/{jobId}** - 查询生成状态
3. **GET /api/game-map/download/{mapId}** - 下载生成的地图

### 数据流

```
用户绘制草图 → 草图数据转换 → AI参数设置 → API请求 → AI处理 → 结果返回 → 显示和下载
```

## 常见问题

### Q: 需要绘制多少图块才能生成好的地图？
A: 建议至少绘制20-30个图块形成基本轮廓。图块越多，AI理解的轮廓越准确。

### Q: 生成一张地图需要多长时间？
A: 根据选择的模型和尺寸，通常需要30-60秒。基础模型较快，专业模型质量更高但需要更长时间。

### Q: 可以生成3D地图吗？
A: 当前版本只支持2D地图生成。3D地图生成功能正在开发中。

### Q: 如何获得更好的生成效果？
A:
1. 绘制清晰的轮廓边界
2. 使用不同图块区分不同区域
3. 选择合适的风格和游戏类型
4. 上传参考图片提供更多视觉线索

### Q: 生成的地图可以商用吗？
A: 使用基础模型和专业模型生成的地图可以用于个人和商业项目。使用定制模型可能需要额外的授权。

## 故障排除

### 问题：生成按钮点击无反应
- 检查是否已绘制草图（至少一个图块）
- 检查所有必填参数是否已选择
- 查看浏览器控制台是否有错误信息

### 问题：生成失败或错误
- 检查网络连接
- 确认API服务是否正常运行
- 查看状态指示器的错误信息

### 问题：下载功能不可用
- 确认生成已完成
- 检查浏览器是否允许下载
- 查看控制台是否有权限错误
