## 常用查询

### 游戏相关查询

#### 1. 查询用户的所有游戏

```sql
SELECT * FROM game
WHERE user_id = 'demo_user_1'
ORDER BY created_at DESC;
```

#### 2. 按类型统计游戏数

```sql
SELECT type, COUNT(*) as count
FROM game
GROUP BY type
ORDER BY count DESC;
```

#### 3. 查询最近创建的游戏

```sql
SELECT * FROM game
ORDER BY created_at DESC
LIMIT 10;
```

#### 4. 查询特定用户最近修改的游戏

```sql
SELECT * FROM game
WHERE user_id = 'demo_user_1'
ORDER BY updated_at DESC
LIMIT 5;
```

#### 5. 统计每个用户的游戏数

```sql
SELECT user_id, COUNT(*) as game_count
FROM game
GROUP BY user_id
ORDER BY game_count DESC;
```

#### 6. 查询游戏统计视图

```sql
SELECT * FROM v_game_statistics
ORDER BY game_count DESC;
```

### AI任务相关查询

#### 1. 查询所有待处理任务

```sql
SELECT * FROM ai_portrait_task
WHERE status = 'PENDING'
ORDER BY created_at ASC;
```

#### 2. 查询某个任务的详细信息

```sql
SELECT * FROM ai_portrait_task
WHERE task_id = 'task_xxx'
LIMIT 1;
```

#### 3. 统计各状态的任务数

```sql
SELECT status, COUNT(*) as count
FROM ai_portrait_task
GROUP BY status;
```

#### 4. 查询失败且可重试的任务

```sql
SELECT * FROM ai_portrait_task
WHERE status = 'FAILED'
  AND retry_count < max_retries
ORDER BY created_at ASC
LIMIT 20;
```

#### 5. 查询今天完成的任务

```sql
SELECT * FROM ai_portrait_task
WHERE DATE(completed_at) = CURDATE()
  AND status = 'SUCCESS'
ORDER BY completed_at DESC;
```

#### 6. 统计任务平均处理时间

```sql
SELECT
    status,
    COUNT(*) as task_count,
    AVG(TIMESTAMPDIFF(SECOND, created_at, completed_at)) as avg_duration_sec,
    MIN(TIMESTAMPDIFF(SECOND, created_at, completed_at)) as min_duration_sec,
    MAX(TIMESTAMPDIFF(SECOND, created_at, completed_at)) as max_duration_sec
FROM ai_portrait_task
WHERE completed_at IS NOT NULL
GROUP BY status;
```

#### 7. 查询任务统计视图

```sql
SELECT * FROM v_ai_task_statistics
WHERE task_date >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
ORDER BY task_date DESC;
```

### AI生成相关查询

#### 1. 查询用户的所有生成记录

```sql
SELECT id, task_id, prompt, model_name, status, created_at
FROM ai_portrait_generation
WHERE user_id = 123
ORDER BY created_at DESC;
```

#### 2. 查询某个模型的使用情况

```sql
SELECT
    model_name,
    COUNT(*) as use_count,
    SUM(CASE WHEN status = 'SUCCESS' THEN 1 ELSE 0 END) as success_count,
    SUM(CASE WHEN status = 'FAILED' THEN 1 ELSE 0 END) as failed_count
FROM ai_portrait_generation
GROUP BY model_name
ORDER BY use_count DESC;
```

#### 3. 查询不同提供商的生成统计

```sql
SELECT
    provider,
    COUNT(*) as generation_count,
    AVG(generation_time) as avg_generation_time,
    MIN(created_at) as first_use,
    MAX(created_at) as last_use
FROM ai_portrait_generation
GROUP BY provider;
```

#### 4. 查询成功率较低的模型

```sql
SELECT
    model_name,
    COUNT(*) as total_count,
    SUM(CASE WHEN status = 'SUCCESS' THEN 1 ELSE 0 END) as success_count,
    ROUND(SUM(CASE WHEN status = 'SUCCESS' THEN 1 ELSE 0 END) / COUNT(*) * 100, 2) as success_rate
FROM ai_portrait_generation
GROUP BY model_name
HAVING success_rate < 90
ORDER BY success_rate ASC;
```

#### 5. 查询最近7天每天的生成数

```sql
SELECT
    DATE(created_at) as date,
    COUNT(*) as generation_count,
    SUM(CASE WHEN status = 'SUCCESS' THEN 1 ELSE 0 END) as success_count,
    AVG(generation_time) as avg_time
FROM ai_portrait_generation
WHERE created_at >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
GROUP BY DATE(created_at)
ORDER BY date DESC;
```

## 性能优化查询

### 1. 查找潜在的长期运行任务

```sql
SELECT
    task_id,
    status,
    retry_count,
    TIMESTAMPDIFF(HOUR, created_at, NOW()) as hours_running
FROM ai_portrait_task
WHERE status = 'PROCESSING'
  AND TIMESTAMPDIFF(HOUR, created_at, NOW()) > 1
ORDER BY created_at ASC;
```

### 2. 检查任务表大小

```sql
SELECT
    table_name,
    ROUND(((data_length + index_length) / 1024 / 1024), 2) AS size_mb
FROM information_schema.TABLES
WHERE table_schema = 'write_engine'
  AND table_name IN ('ai_portrait_task', 'ai_portrait_generation', 'operation_log')
ORDER BY (data_length + index_length) DESC;
```

### 3. 查找频繁失败的操作

```sql
SELECT
    DATE(created_at) as date,
    operation_type,
    COUNT(*) as fail_count
FROM operation_log
WHERE status = 'FAILED'
  AND created_at >= DATE_SUB(NOW(), INTERVAL 7 DAY)
GROUP BY DATE(created_at), operation_type
ORDER BY date DESC, fail_count DESC;
```

### 4. 查找热点用户

```sql
SELECT
    user_id,
    COUNT(*) as operation_count,
    COUNT(DISTINCT operation_type) as operation_types
FROM operation_log
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY user_id
ORDER BY operation_count DESC
LIMIT 20;
```

## 常见问题排查

### 问题1: 查询超时

**症状**: 某些查询执行缓慢

**排查步骤**:

```sql
-- 1. 检查是否有全表扫描
EXPLAIN SELECT * FROM ai_portrait_generation WHERE prompt LIKE '%xxx%';

-- 2. 检查表索引
SHOW INDEX FROM ai_portrait_generation;

-- 3. 优化查询
-- 添加合适的索引
CREATE INDEX idx_prompt ON ai_portrait_generation(prompt(100));
```

### 问题2: 磁盘空间满

**症状**: 数据库停止写入

**排查步骤**:

```sql
-- 1. 查看表大小
SELECT table_name, ROUND(((data_length + index_length) / 1024 / 1024), 2) AS size_mb
FROM information_schema.TABLES
WHERE table_schema = 'write_engine'
ORDER BY (data_length + index_length) DESC;

-- 2. 清理过期数据
CALL sp_cleanup_old_tasks(30);

-- 3. 清理日志
DELETE FROM operation_log WHERE DATE(created_at) < DATE_SUB(CURDATE(), INTERVAL 90 DAY);
```

### 问题3: 死锁

**症状**: 事务互相等待

**排查步骤**:

```sql
-- 1. 查看锁信息
SHOW ENGINE INNODB STATUS;

-- 2. 查看当前锁
SELECT * FROM INFORMATION_SCHEMA.INNODB_LOCKS;

-- 3. 查看锁等待
SELECT * FROM INFORMATION_SCHEMA.INNODB_LOCK_WAITS;
