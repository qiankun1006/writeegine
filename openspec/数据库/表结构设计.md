## 索引优化策略

### 复合索引

用于常用查询的复合索引优化：

```sql
-- 游戏表
CREATE INDEX idx_game_user_type ON game(user_id, type);
CREATE INDEX idx_game_created_updated ON game(created_at, updated_at);

-- AI任务表
CREATE INDEX idx_ai_task_status_created ON ai_portrait_task(status, created_at);

-- AI生成表
CREATE INDEX idx_ai_generation_user_status ON ai_portrait_generation(user_id, status);
```

### 索引使用建议

1. **避免全表扫描**: 确保WHERE子句中的字段有索引
2. **避免索引失效**: 不要在索引字段上进行函数操作
3. **定期分析**: 使用 `ANALYZE TABLE` 更新表统计信息
4. **监控性能**: 定期检查慢查询日志

## 字符集和排序规则

所有表使用以下配置：

- **字符集**: `utf8mb4` - 支持完整的Unicode字符集（包括emoji）
- **排序规则**: `utf8mb4_unicode_ci` - Unicode排序，不区分大小写

```sql
CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
```

## 扩展建议

1. **分表策略**: 当操作日志表或任务表数据量过大时，考虑按时间分表
2. **归档策略**: 定期归档已完成的任务数据到历史表
3. **备份策略**: 每天备份一次，保留最近30天的备份
4. **监控告警**: 监控表大小、慢查询、锁等待等指标
