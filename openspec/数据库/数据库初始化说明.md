## 🔧 常见问题

### Q1: 执行脚本时出现权限错误？
**A:** 确保你的 MySQL 用户拥有 `CREATE DATABASE` 权限
```bash
mysql -u root -p -e "GRANT ALL PRIVILEGES ON *.* TO 'root'@'localhost';"
```

### Q2: MySQL 连接超时？
**A:** 检查 MySQL 是否正在运行
```bash
# macOS/Linux
sudo systemctl status mysql

# 或使用 Homebrew
brew services list | grep mysql
```

### Q3: 表已存在错误？
**A:** 这是正常的，脚本使用 `CREATE TABLE IF NOT EXISTS` 会跳过已存在的表

### Q4: 字符编码问题？
**A:** 脚本已指定 UTF8MB4 编码，如有问题可以在 MySQL 配置文件中设置：
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
```

## 🛠️ 维护建议

### 定期备份
```bash
# 每天备份数据库
mysqldump -u root -p writeengine > writeengine_backup_$(date +%Y%m%d).sql
```

### 清理过期任务
```sql
-- 使用存储过程清理 30 天以前的过期任务
CALL sp_cleanup_old_tasks(30);
```

### 查看数据库大小
```sql
SELECT
  TABLE_NAME,
  ROUND(((DATA_LENGTH + INDEX_LENGTH) / 1024 / 1024), 2) as Size_MB
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_SCHEMA = 'writeengine';
```
