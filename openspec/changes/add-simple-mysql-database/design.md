## Context
实现一个简易MySQL数据库系统，包含现代关系型数据库的核心特性。该项目旨在深入理解数据库内部机制，包括存储、索引、事务、并发控制等关键技术。项目基于Spring Boot框架，使用Java语言开发。

目标用户是希望学习数据库内部实现的开发者，以及需要轻量级数据库解决方案的应用场景。

## Goals / Non-Goals

### Goals
- 实现SQL解析器，支持基本的CREATE TABLE、INSERT、SELECT、UPDATE、DELETE语句
- 实现基于规则和成本的查询优化器
- 实现页式存储引擎，支持B+树聚簇索引
- 实现支持ACID特性的事务管理系统
- 实现MVCC多版本并发控制机制
- 实现Undo Log、Redo Log和Binary Log日志系统
- 实现缓冲池管理，包含LRU淘汰策略
- 实现行级锁和表级锁管理
- 实现基础的MySQL网络协议支持
- 提供性能监控和统计信息

### Non-Goals
- 完全兼容MySQL的所有SQL语法和函数
- 分布式存储和集群支持
- 复杂的查询优化算法（如动态规划优化器）
- 完整的权限和安全管理系统
- 生产级的高可用和故障恢复

## Decisions

### 架构设计决策
- **分层架构**: 采用经典的数据库分层架构
  - 网络层: 处理客户端连接和协议解析
  - SQL层: SQL解析、查询优化、执行计划
  - 存储层: 存储引擎、缓冲池、索引管理
  - 事务层: 事务管理、锁管理、日志管理

### 存储引擎设计
- **页式存储**: 使用固定大小页面（16KB），类似InnoDB
- **B+树索引**: 聚簇索引存储数据，二级索引存储主键引用
- **文件组织**: 每个表对应一个.ibd文件，包含数据页和索引页

### 事务管理设计
- **隔离级别**: 支持READ COMMITTED和REPEATABLE READ
- **MVCC实现**: 基于undo log链实现多版本读
- **锁策略**: 行级锁 + 意向锁，避免死锁

### 日志系统设计
- **Redo Log**: WAL机制，保证持久性
- **Undo Log**: 支持回滚和MVCC读
- **Binary Log**: 记录数据变更，支持复制（简化版）

### 技术栈选择
- **Java 1.8**: 兼容性和性能
- **Spring Boot**: 提供依赖注入和配置管理
- **NIO**: 高性能网络IO处理
- **内存映射文件**: 高效的磁盘IO

## Risks / Trade-offs

### 性能风险
- **风险**: Java GC可能影响数据库性能
- **缓解**: 使用堆外内存存储关键数据结构，优化对象分配

### 并发控制复杂性
- **风险**: 死锁检测和处理逻辑复杂
- **缓解**: 实现简单的死锁检测算法，超时回滚机制

### 存储可靠性
- **风险**: 数据损坏和丢失
- **缓解**: 实现页面校验和，定期数据一致性检查

### 内存使用
- **风险**: 缓冲池内存使用过多
- **缓解**: 实现LRU淘汰策略，支持内存使用限制配置
