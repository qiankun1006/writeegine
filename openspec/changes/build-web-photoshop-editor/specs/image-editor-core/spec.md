# 图片编辑器核心架构规范

## ADDED Requirements

### Requirement: 事件系统架构
系统SHALL提供一个事件总线（EventBus）用于组件间的通信，支持事件发布/订阅模式。

#### Scenario: 工具切换事件通知所有监听者
- **WHEN** 用户切换绘图工具
- **THEN** 事件系统发送 `toolChanged` 事件，所有注册的监听者（如UI面板、属性编辑器）都收到通知并更新状态

#### Scenario: 文档改变事件触发画布刷新
- **WHEN** 绘制、编辑或调整操作完成
- **THEN** 系统发送 `documentChanged` 事件，画布渲染器订阅此事件并重新渲染

### Requirement: 图层数据模型
系统SHALL定义完整的图层数据结构，支持图层的基本操作和属性管理。

#### Scenario: 创建和管理多个图层
- **WHEN** 用户点击"新建图层"按钮
- **THEN** 创建新图层，可设置名称、调整不透明度、改变混合模式、显示/隐藏

#### Scenario: 图层重新排序
- **WHEN** 用户进行图层拖拽或点击上移/下移按钮
- **THEN** 图层堆叠顺序改变，画布重新渲染

### Requirement: 撤销/重做系统
系统SHALL实现完整的撤销/重做功能，支持所有编辑操作。

#### Scenario: 撤销绘制操作
- **WHEN** 用户点击撤销按钮或按 Ctrl+Z
- **THEN** 回退到上一个编辑状态

#### Scenario: 重做操作
- **WHEN** 用户点击重做按钮或按 Ctrl+Y
- **THEN** 恢复到下一个编辑状态（如果存在）

#### Scenario: 历史记录链长度有限
- **WHEN** 执行超过100个命令后
- **THEN** 最早的命令自动删除，最多保留100条历史

### Requirement: 工具插件框架
系统SHALL提供可扩展的工具框架，支持添加新工具而无需修改核心代码。

#### Scenario: 注册新工具
- **WHEN** 开发者创建继承自Tool基类的工具并注册到工具管理器
- **THEN** 新工具可被激活和使用

#### Scenario: 工具选项动态更新
- **WHEN** 用户切换不同工具
- **THEN** UI动态显示该工具对应的选项面板

### Requirement: 基础渲染引擎
系统SHALL实现Canvas渲染器，支持图层合成、视口变换和高效重绘。

#### Scenario: 多图层合成渲染
- **WHEN** 文档包含多个图层
- **THEN** 系统按正确的堆叠顺序混合，应用不透明度和混合模式

#### Scenario: 视口缩放
- **WHEN** 用户滚动鼠标滚轮
- **THEN** 画布放大或缩小，保持缩放中心不变

#### Scenario: 视口平移
- **WHEN** 用户拖拽画布或使用空格+拖拽
- **THEN** 画布移动，展示不同区域

### Requirement: 基础选择工具
系统SHALL实现矩形和椭圆选择工具，支持基本的选区操作。

#### Scenario: 矩形选择
- **WHEN** 用户激活矩形选择工具并在画布上拖拽
- **THEN** 绘制矩形选区，显示虚线边界

#### Scenario: 椭圆选择
- **WHEN** 用户激活椭圆选择工具并在画布上拖拽
- **THEN** 绘制椭圆选区，显示虚线边界

### Requirement: 基础绘制工具
系统SHALL实现画笔和铅笔工具，支持实时绘制。

#### Scenario: 画笔绘制
- **WHEN** 用户激活画笔工具并在画布上拖拽
- **THEN** 以柔和边缘笔刷绘制到当前图层

#### Scenario: 铅笔绘制
- **WHEN** 用户激活铅笔工具并在画布上拖拽
- **THEN** 以硬边缘笔刷绘制到当前图层

### Requirement: 基础颜色调整
系统SHALL实现亮度/对比度调整功能。

#### Scenario: 调整亮度和对比度
- **WHEN** 用户调整亮度或对比度滑块
- **THEN** 实时预览效果，确认后应用到图层，支持撤销/重做

### Requirement: 页面路由与UI
系统SHALL提供图片编辑页面的路由和完整的UI布局。

#### Scenario: 访问图片编辑器
- **WHEN** 用户从 `/create-game` 点击"图片编辑"链接
- **THEN** 跳转到 `/create-game/image`，加载图片编辑器

#### Scenario: 编辑器UI布局
- **WHEN** 编辑器页面加载
- **THEN** 显示工具栏、画布区域、图层面板、工具选项面板、历史记录面板

### Requirement: 文档保存与导出
系统SHALL支持保存编辑文档和导出为标准图片格式。

#### Scenario: 保存文档
- **WHEN** 用户点击保存按钮
- **THEN** 将编辑的文档保存到服务器，包括所有图层和编辑历史

#### Scenario: 导出为PNG格式
- **WHEN** 用户点击导出PNG按钮
- **THEN** 生成PNG格式文件并下载到本地

#### Scenario: 导出为JPG格式
- **WHEN** 用户点击导出JPG按钮
- **THEN** 显示质量选项，生成JPG文件并下载到本地
