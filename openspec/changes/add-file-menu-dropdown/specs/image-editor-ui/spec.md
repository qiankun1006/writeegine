# 规范：图片编辑器 UI - 文件菜单下拉功能

## ADDED Requirements

### Requirement: 文件菜单下拉显示

系统 SHALL 在用户点击"文件"按钮时显示一个下拉菜单，包含常用的文件操作选项。菜单 SHALL 在用户点击菜单外区域时自动关闭。

#### Scenario: 用户点击文件菜单按钮
- **WHEN** 用户点击菜单栏的"文件"按钮
- **THEN** 一个下拉菜单出现，显示"导入、保存、另存为、导出为 PNG、导出为 JPG"等选项
- **AND** 菜单使用平滑的过渡动画显示

#### Scenario: 用户点击菜单外区域
- **WHEN** 用户在菜单打开时点击菜单外的任何区域
- **THEN** 下拉菜单立即关闭

#### Scenario: 用户再次点击文件菜单按钮
- **WHEN** 菜单已打开，用户再次点击"文件"按钮
- **THEN** 下拉菜单关闭

### Requirement: 导入图片文件

系统 SHALL 允许用户从本地磁盘选择图片文件并导入到编辑器中进行编辑。

#### Scenario: 用户选择导入图片
- **WHEN** 用户点击菜单中的"导入"选项
- **THEN** 浏览器文件选择对话框打开，允许用户选择图片或文档文件
- **AND** 用户选择文件后，该文件被加载到编辑器中

#### Scenario: 用户导入新文档文件
- **WHEN** 用户选择一个 JSON 格式的编辑文档文件
- **THEN** 该文档的内容（包括所有图层和编辑历史）被加载到编辑器

#### Scenario: 用户导入图片文件
- **WHEN** 用户选择一个 PNG 或 JPG 图片文件
- **THEN** 图片被作为新的内容加载到编辑器的画布上

### Requirement: 保存编辑文档

系统 SHALL 允许用户将当前编辑的文档保存为本地文件，以便后续继续编辑。

#### Scenario: 用户保存当前文档
- **WHEN** 用户点击菜单中的"保存"选项
- **THEN** 当前编辑的文档被导出为 JSON 格式的文件，文件名为 `document.json`
- **AND** 浏览器的下载功能被触发，用户可以保存该文件

#### Scenario: 保存应包含完整的编辑信息
- **WHEN** 文档被保存
- **THEN** 保存的 JSON 文件应包括所有图层、编辑历史、工具设置等信息

### Requirement: 另存为编辑文档

系统 SHALL 允许用户指定新的文件名来保存当前文档。

#### Scenario: 用户另存为文档
- **WHEN** 用户点击菜单中的"另存为"选项
- **THEN** 浏览器弹出输入框，提示用户输入新的文件名
- **AND** 用户输入文件名后，文档以新文件名导出

#### Scenario: 用户取消另存为操作
- **WHEN** 用户在输入框中点击取消或关闭对话框
- **THEN** 不执行任何操作，菜单关闭

### Requirement: 导出为 PNG 图片

系统 SHALL 允许用户将当前编辑的内容导出为 PNG 格式的图片文件。

#### Scenario: 用户导出为 PNG
- **WHEN** 用户点击菜单中的"导出为 PNG"选项
- **THEN** 编辑器当前画布的内容被转换为 PNG 图片
- **AND** 文件名为 `export.png` 的文件被下载到用户的本地磁盘

#### Scenario: 导出内容包括所有可见元素
- **WHEN** 图片被导出
- **THEN** 导出的 PNG 文件包括所有可见的图层、绘制内容和效果

### Requirement: 导出为 JPG 图片

系统 SHALL 允许用户将当前编辑的内容导出为 JPG 格式的图片文件。

#### Scenario: 用户导出为 JPG
- **WHEN** 用户点击菜单中的"导出为 JPG"选项
- **THEN** 编辑器当前画布的内容被转换为 JPG 图片
- **AND** 文件名为 `export.jpg` 的文件被下载到用户的本地磁盘

#### Scenario: 导出内容质量合理
- **WHEN** 图片被导出为 JPG
- **THEN** 导出的文件使用合理的压缩质量（建议 0.85），确保文件大小与质量的平衡

### Requirement: 菜单系统扩展基础

系统 SHALL 提供可复用的菜单管理架构，便于未来添加其他菜单（编辑、图像、图层、选择、滤镜、查看）。

#### Scenario: 菜单管理系统的设计
- **WHEN** 实现文件菜单功能
- **THEN** 应该创建一个可复用的 `MenuManager` 基类或模式
- **AND** 该模式应该支持菜单的打开/关闭、项目点击处理、外部点击关闭等通用功能

#### Scenario: 其他菜单的集成
- **WHEN** 未来需要实现其他菜单
- **THEN** 可以基于现有的菜单系统快速实现，无需重复编写菜单逻辑
