/openspec/proposal   需求：基于 Vue3 + Composition API 开发前端进度条组件，集成到 http://localhost:8083/create-game/asset页面下，新增一个功能块，具体要求如下：

1. 核心技术框架
   - 基于 Vue3 (Setup 语法糖 + Composition API) 开发，支持 Vue3 最新特性（如 ref/reactive/watch）；
   - 组件需封装为独立的 .vue 文件，支持 Vite 构建，无第三方 UI 库强依赖（基础样式用原生 CSS3 实现，可选兼容 Element Plus/Naive UI）；


2. 多风格/多效果支持（核心）
   内置以下 6 种可切换的进度条效果（覆盖游戏场景常用样式）：
   - 效果1：极简纯色款（无纹理）→ 支持自定义背景色、填充色、圆角、边框；
   - 效果2：游戏血条款 → 支持线性渐变（如深红→鲜红）、血滴纹理、受损闪烁动画、血量低于20%时的警示抖动效果；
   - 效果3：能量条款 → 支持径向渐变（如蓝色→紫色）、流光动画、充能完成时的脉冲效果；
   - 效果4：加载进度款 → 支持条纹动效（CSS 动画）、进度数字实时显示、加载完成/失败的状态图标；
   - 效果5：复古像素款 → 8bit 像素风格边框、阶梯式填充动画、像素字体进度显示；
   - 效果6：玻璃拟态款 → 毛玻璃背景、半透明填充、模糊边框，适配现代游戏UI风格。

3. 血条专属编辑能力
   - 支持两种编辑方式：① 拖拽进度条滑块调整血量值；② 输入框手动输入 0-100 数值，编辑后实时同步到接口参数；
   - 血条参数可配置：渐变起始色/结束色、是否显示血量数值（HP）、是否启用低血量警示、边框像素大小（像素款）；
   - 编辑后的血条样式可预览，支持一键保存配置到 /create-game/asset 接口。

4. 交互与动画要求
   - 所有进度条效果均支持平滑过渡动画（进度更新无卡顿），动画速度可配置（0.2s-1s 可调）；
   - 悬停时显示 tooltip，内容包含「当前进度 + 关联资产名称 + 接口状态」；
   - 支持进度条状态切换：加载中/完成/失败，不同状态对应不同视觉反馈（如失败变红+闪烁、完成变绿+对勾图标）；
   - 响应式适配：在PC端（游戏编辑器）、平板端均能正常显示，进度条宽度/高度支持百分比/固定像素配置。

5. 输出要求
   请输出以下内容，确保代码可直接运行：
   1) 完整的 ProgressBar.vue 组件代码（含 template/style/script setup）；
   2) 组件使用示例（绑定 /create-game/asset 接口的调用逻辑，含异步请求、进度更新、状态处理）；
   3) 效果切换/参数配置的 API 文档（如 props 说明、事件说明）；
   4) 样式自定义示例（如如何新增自定义进度条效果）；
   5) 注意事项（如浏览器兼容性、性能优化建议）。
