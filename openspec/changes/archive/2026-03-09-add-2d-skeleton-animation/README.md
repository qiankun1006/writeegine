## 🔧 技术选型

### 推荐方案：从零实现 FK 系统（优化长期维护）

**优点：**
- 完全可控，代码清晰
- 与现有系统无缝集成
- 易于扩展到 IK、粒子等高级功能

**核心算法：**
- 正向动力学（FK）: 矩阵变换，父子级联
- 插值: 线性、贝塞尔、步进
- 碰撞检测: 可选，后期添加

### 备选方案：集成现成库（快速上线）

**可选库：**
- DragonBones (Flash-style bone animation)
- Spine (Professional 2D skeletal animation)
- Pixi.js 的骨骼动画支持

**使用场景：** 如果要快速发布 v1.0，可考虑集成库，后续逐步自研

## ❓ 常见问题

### Q: 为什么选择 FK 而不是 IK？
**A:** FK 是骨骼动画的基础，易于实现且高效。IK 作为高级功能，可在 v2.0 中添加。

### Q: 能否集成现成的骨骼动画库？
**A:** 可以，但需要权衡快速上线 vs 长期维护。建议先自研 MVP，验证需求后再决定。

### Q: 多长时间能完成？
**A:**
- MVP (基础骨骼 + 关键帧): 2 周
- 完整功能: 4 周
- 优化和文档: 1 周

### Q: 如何与现有编辑器集成？
**A:** 通过扩展 Layer、Tool、Command、Menu 系统。详见 design.md 的"集成方案"部分。
