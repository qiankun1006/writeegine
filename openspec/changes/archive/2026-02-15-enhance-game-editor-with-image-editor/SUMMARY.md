# 提案总结：扩展创作游戏页面 - 添加图片编辑功能入口

## 📌 提案概览

本提案计划扩展现有的【创作游戏】页面，从单个 Tilemap 编辑器改造为支持多编辑器的统一编辑中心。核心改造是引入**选项卡界面**，让用户可以在「地图编辑」和「图片编辑」之间无缝切换。

**阶段目标：** 搭建选项卡框架，为图片编辑器预留入口（具体实现后续进行）

---

## 📋 主要改造

### 页面结构
```
原结构：
├── 游戏信息表单
├── Tilemap 编辑器
└── 操作按钮

新结构：
├── 游戏信息表单
├── 编辑器选项卡
│   ├── Tab 1: 地图编辑 (Tilemap)
│   └── Tab 2: 图片编辑 (占位符)
└── 操作按钮
```

## 🎨 用户界面设计

### 选项卡布局
```
┌─ 🗺️ 地图编辑 [活跃] │ 🖼️ 图片编辑 ─┐
├──────────────────────────────────┤
│                                  │
│  Tilemap 编辑器                   │
│  (Canvas、工具栏、网格等)         │
│                                  │
└──────────────────────────────────┘
```

### 选项卡状态

| 状态 | 样式 | 说明 |
|------|------|------|
| 活跃 | 高亮 + 下划线 | 当前选中的编辑器 |
| 非活跃 | 灰色 | 未选中的编辑器 |
| 悬停 | 浅蓝 | 鼠标悬停时 |

---

## 🔧 技术方案

### 1. HTML 改造
- 使用 `<div class="editor-tabs">` 包装所有编辑器
- 使用 `<button class="editor-tab-button">` 作为选项卡按钮
- 使用 `<div class="editor-tab-content">` 作为内容容器
- 保留 `th:replace` 嵌入 Tilemap 编辑器

### 2. CSS 样式
- 选项卡按钮使用 Flexbox 布局
- 活跃状态通过 `.active` 类控制
- 使用 `@keyframes fadeIn` 实现淡入淡出动画
- 在 768px 断点处调整移动端样式

### 3. JavaScript 交互
- 创建 `EditorTabManager` 类
- 管理选项卡的活跃状态
- 处理编辑器的初始化回调
- 预留接口供后续图片编辑器使用

## ❓ 常见问题

### Q: 为什么选择选项卡而不是侧边栏？
A: 选项卡在编辑工具中是标准做法，用户易于理解。侧边栏会占用水平空间，不利于编辑器的全屏展示。

### Q: 图片编辑器什么时候开发？
A: 本提案仅搭建框架和入口，具体的图片编辑器开发在后续迭代进行。

### Q: 如何在两个编辑器间切换用户创建的内容？
A: 后续可以在 EditorTabManager 中添加数据同步机制，或使用全局状态管理。

### Q: 会影响现有的编辑功能吗？
A: 不会。所有改造都是向后兼容的，现有功能保持不变。
