# 任务列表：游戏创作平台门户重构

## 项目概览

将现有的通用 Web 应用首页重构为游戏创作平台的专业门户，添加三个核心功能入口，并将 Tilemap 编辑器集成为创作游戏的子模块。

---

## 第一阶段：前端基础设施

### T1.1：首页门户设计与实现
- [ ] 设计首页布局（导航栏、英雄区、菜单卡片、页脚）
- [ ] 编写 `src/main/resources/templates/index.html`
- [ ] 创建深色科技感主题 CSS (`src/main/resources/static/css/homepage.css`)
- [ ] 实现响应式设计（适配 mobile/tablet/desktop）
- [ ] 添加交互增强 JavaScript (`src/main/resources/static/js/homepage.js`)
- 验证：页面能正常加载，所有卡片可点击，导航正常

### T1.2：创作游戏页面骨架
- [ ] 编写 `src/main/resources/templates/create-game.html` 骨架
- [ ] 创建游戏信息表单（名称、描述等）
- [ ] 预留 Tilemap 编辑器容器位置
- [ ] 编写相关 CSS (`src/main/resources/static/css/game-creation.css`)
- 验证：页面能正常加载，表单可交互

### T1.3：我的游戏列表页面
- [ ] 编写 `src/main/resources/templates/my-games.html`
- [ ] 设计游戏列表展示（缩略图、标题、描述、操作按钮）
- [ ] 实现空状态提示
- [ ] 编写相关 CSS (`src/main/resources/static/css/game-management.css`)
- 验证：页面能正常加载，列表布局正确

### T1.4：游戏广场页面
- [ ] 编写 `src/main/resources/templates/game-plaza.html`
- [ ] 设计游戏网格展示（卡片样式）

- [ ] 编写相关 CSS (`src/main/resources/static/css/game-plaza.css`)

## 第二阶段：后端功能

### T2.1：路由配置更新
- [ ] 更新 `HomeController.java`，新增四个路由：
  - `GET /` (首页)
  - `GET /create-game` (创作游戏)
  - `GET /my-games` (我的游戏)
  - `GET /game-plaza` (游戏广场)
- [ ] 保留 `GET /about` 以向后兼容
- [ ] 保留 `GET /tilemap-editor` 以向后兼容（或改为 redirect）
- 验证：所有路由能正常访问，返回正确的视图

### T2.2：游戏实体模型（可选/后期）
- [ ] 定义 Game 实体 (id, name, description, author, createdAt, updatedAt, status, etc.)
- [ ] 创建 GameRepository 接口
- [ ] 创建 GameService 业务逻辑
- [ ] 实现基本的 CRUD 操作
- 验证：实体能正常持久化和查询

### T2.3：数据绑定
- [ ] 在 `HomeController` 中注入 GameService
- [ ] 在 `createGame()` 方法中加载或初始化游戏数据
- [ ] 在 `myGames()` 方法中加载用户游戏列表
- [ ] 在 `gamePlaza()` 方法中加载已发布游戏列表
- 验证：页面能正确展示动态数据
