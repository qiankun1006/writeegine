## MODIFIED Requirements

### Requirement: 响应式三栏布局

**修改为双栏简化布局**。系统 SHALL 在 PC 端（≥1200px）提供清晰的左右分栏布局，左侧集中参数管理，右侧专注结果展示。

#### Scenario: 左右分栏布局
- **WHEN** 用户在 1200px 及以上宽度屏幕上访问 AI 立绘生成器
- **THEN** 页面显示左右分栏布局：
  - 左侧：控制区（固定宽度 380px）
  - 右侧：预览区（自适应填充剩余宽度）
- **AND** 左右间距为 20px
- **AND** 整体内边距为 20px
- **AND** 左侧控制区支持垂直滚动（`overflow-y: auto`）

#### Scenario: 左侧控制区模块化
- **WHEN** 用户看到左侧控制面板
- **THEN** 面板包含三个有序的模块：
  1. 提示词模块（始终展开）
  2. 参考图片模块（始终展开）
  3. 参数配置模块（默认折叠）
- **AND** 各模块之间有适当的间距分隔
- **AND** 模块标题清晰，字体大小一致

#### Scenario: 右侧预览区分层
- **WHEN** 用户看到右侧预览区
- **THEN** 预览区分为两层：
  - 上层（占 90% 高度）：预览内容区，虚线边框，默认显示"点击生成查看结果"提示文字
  - 下层（占 10% 高度）：操作按钮区，包含"生成图片"（主按钮）和"历史记录"（文字按钮）

## ADDED Requirements

### Requirement: 提示词输入模块

系统 SHALL 在左侧控制区提供两个多行文本输入框，分别用于正面提示词和负面提示词的输入。

#### Scenario: 提示词框初始化
- **WHEN** 页面加载时
- **THEN** 显示两个输入框：
  - 正面提示词框：占位符"例如：日系二次元少女，长发粉发，猫耳，穿洛丽塔裙，纯色背景，光影柔和"
  - 负面提示词框：占位符"例如：模糊，多手指，水印，变形，低分辨率"
- **AND** 两个输入框都支持多行文本输入

#### Scenario: 提示词输入交互
- **WHEN** 用户在提示词框中输入内容
- **THEN** 输入框显示聚焦时的发光效果（边框高亮）
- **AND** 支持长按 Ctrl/Cmd + A 全选
- **AND** 支持粘贴功能

### Requirement: 参考图片上传模块

系统 SHALL 提供单张图片上传功能，支持拖拽和点击上传，上传后显示缩略图预览，支持删除操作。

#### Scenario: 上传区域初始状态
- **WHEN** 页面加载时未上传图片
- **THEN** 显示一个上传区域
- **AND** 区域内显示上传图标和提示文字"拖拽图片或点击选择"
- **AND** 边框为虚线风格，背景色为浅灰

#### Scenario: 拖拽上传图片
- **WHEN** 用户将图片文件拖拽到上传区域
- **THEN** 上传区域边框高亮，背景色变深
- **AND** 释放后图片开始上传
- **AND** 显示上传进度

#### Scenario: 文件验证与缩略图显示
- **WHEN** 用户选择或上传完成一张图片
- **THEN** 系统验证：
  - 文件格式：仅允许 PNG、JPG、WEBP
  - 文件大小：≤10MB
- **AND** 验证成功时显示缩略图预览（200×200 像素）
- **AND** 缩略图上方显示删除按钮（❌ 图标）

#### Scenario: 删除已上传图片
- **WHEN** 用户点击缩略图上的删除按钮
- **THEN** 图片被移除
- **AND** 上传区域恢复到初始空状态

### Requirement: 参数配置模块（折叠设计）

系统 SHALL 在左侧控制区提供一个可折叠的参数配置模块，包含基本参数设置，默认处于收起状态。

#### Scenario: 参数模块初始化
- **WHEN** 页面加载时
- **THEN** 参数配置模块显示标题"⚙️ 高级参数"
- **AND** 标题右侧显示展开箭头（向右 >）
- **AND** 模块处于收起状态，内容隐藏

#### Scenario: 展开参数配置
- **WHEN** 用户点击参数模块标题或箭头
- **THEN** 模块平滑展开（动画时间 0.3s）
- **AND** 显示以下配置项：
  1. 模型选择（下拉框）：SD 1.5 / SDXL / Realistic，默认 SD 1.5
  2. 图生图强度（滑块）：范围 0-1，默认 0.5，实时显示值
  3. 采样器选择（下拉框）：Euler / DPM++ / Auto，默认 Euler
  4. 迭代步数（数字输入框）：默认 20，范围 10-50
- **AND** 箭头向下旋转 180°

#### Scenario: 收起参数配置
- **WHEN** 用户再次点击参数模块标题
- **THEN** 模块平滑收起（动画时间 0.3s）
- **AND** 内容隐藏，仅显示标题
- **AND** 箭头恢复向右

#### Scenario: 参数项样式与交互
- **WHEN** 查看参数配置中的各个控件
- **THEN** 每个参数项包含：
  - 参数标签（左对齐）
  - 参数输入控件（占据右侧空间）
  - Hover 时显示说明 Tooltip
- **AND** 所有控件圆角 8px，hover 时边框高亮

### Requirement: 生成图片按钮与加载状态

系统 SHALL 提供主生成按钮，支持加载状态显示，点击后模拟 3 秒生成，然后显示示例图片。

#### Scenario: 按钮初始状态
- **WHEN** 页面加载或生成完成后
- **THEN** "生成图片"按钮显示在右侧操作区下层
- **AND** 按钮为启用状态，主色调 #409EFF（蓝色）
- **AND** 按钮文字为"✨ 生成图片"
- **AND** 按钮尺寸宽度占操作区 50-70%，高度 40-44px

#### Scenario: 按钮点击后加载状态
- **WHEN** 用户点击"生成图片"按钮
- **THEN** 按钮立即禁用，不可再次点击
- **AND** 按钮文字变为"⏳ 生成中..."
- **AND** 按钮内显示加载动画（旋转的 icon 或进度条）
- **AND** 按钮背景色变暗，表示禁用状态

#### Scenario: 模拟生成过程
- **WHEN** 用户点击生成按钮后
- **THEN** 系统模拟 3 秒的生成过程
- **AND** 3 秒后显示示例图片在上方预览区
- **AND** 按钮恢复到初始状态，可再次点击

#### Scenario: 预览区默认提示
- **WHEN** 页面加载或未生成图片时
- **THEN** 预览区中央显示文字提示"点击生成查看结果"
- **AND** 文字颜色为浅灰，字体大小 16px

#### Scenario: 图片显示
- **WHEN** 生成完成后
- **THEN** 示例图片在预览区居中显示
- **AND** 图片宽度自适应预览区，不超过容器宽度
- **AND** 图片以淡入动画出现（0.5s）

### Requirement: 代码结构与技术栈

系统 SHALL 采用 Vue 3 现代语法和 Element Plus UI 组件库，代码结构清晰、可维护性强。

#### Scenario: 脚本语言与语法
- **WHEN** 查看源代码
- **THEN** 所有组件使用 Vue 3 `<script setup>` 语法
- **AND** 类型支持：使用 TypeScript（.ts 扩展名）或普通 JavaScript
- **AND** 变量声明使用 `const`/`let`，响应式数据使用 `ref()`/`reactive()`

#### Scenario: UI 组件库集成
- **WHEN** 检查组件实现
- **THEN** 使用 Element Plus 组件库提供的组件：
  - `ElInput` - 文本输入框
  - `ElUpload` - 文件上传
  - `ElButton` - 按钮
  - `ElCollapse` - 折叠面板（用于参数配置）
  - `ElSelect` - 下拉选择框
  - `ElSlider` - 滑块控件
- **AND** 正确导入并使用各组件

#### Scenario: 代码注释与结构
- **WHEN** 查看组件源代码
- **THEN** 代码应包含清晰的注释：
  - 顶部有功能说明注释
  - 关键业务逻辑前有说明
  - 复杂计算或条件判断有说明
- **AND** 代码使用分层结构：
  - 导入区（imports）
  - 组件选项（props、emits 等）
  - 响应式数据定义
  - 计算属性和方法
  - 生命周期钩子

#### Scenario: 样式封装
- **WHEN** 查看 `<style>` 块
- **THEN** 所有样式使用 `scoped` 属性隔离
- **AND** 使用 SCSS 预处理器编写（`lang="scss"`）
- **AND** 从 `@/styles/theme.scss` 导入主题变量
- **AND** 避免全局样式污染
- **AND** 使用 CSS 变量或 SCSS 变量管理颜色和间距

## REMOVED Requirements

### Requirement: 响应式三栏/四栏布局

**移除复杂的多栏响应式布局**。原有的三栏（宽屏）和四栏（超宽屏）设计将被简化为统一的左右分栏布局。

**理由**：简化设计，降低用户认知负担，提升核心流程体验。
**迁移**：所有高级参数功能保留在左侧折叠模块中，继续可用。
