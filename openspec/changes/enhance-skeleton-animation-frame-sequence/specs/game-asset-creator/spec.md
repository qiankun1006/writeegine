# Spec: 游戏素材创建器 - 动画帧序列集成

## MODIFIED Requirements

### 页面菜单结构
#### Requirement: 游戏素材创作页面菜单
游戏素材创作页面必须提供完整的素材类型分类菜单，包括角色相关、地图与场景、UI界面类、特效与动画、文字与图标、剧情与过场等分类。

#### Scenario: 用户访问素材创作页面
- **WHEN** 用户访问 `http://localhost:8083/create-game/asset`
- **THEN** 显示左侧分类菜单，包含6个主要分类
- **AND** 【一、角色相关】分类下包含7个菜单项，包括新增的【动画帧序列】
- **AND** 点击【动画帧序列】显示完整的骨骼动画编辑器

#### Scenario: 菜单项切换
- **WHEN** 用户点击【动画帧序列】菜单项
- **THEN** 隐藏其他编辑器面板，显示动画帧序列编辑器
- **AND** 编辑器包含画布、时间轴、骨骼树、属性面板等组件
- **AND** 编辑器自动初始化并加载默认项目

### 编辑器集成
#### Requirement: 多编辑器面板管理
系统必须支持在同一个页面中管理多个不同类型的编辑器面板，并实现平滑切换。

#### Scenario: 编辑器面板切换
- **WHEN** 用户从【动画帧序列】切换到【角色立绘】编辑器
- **THEN** 动画帧序列编辑器保存当前状态并隐藏
- **AND** 角色立绘编辑器加载并显示
- **AND** 切换过程流畅，无页面刷新

#### Scenario: 编辑器状态保持
- **WHEN** 用户在【动画帧序列】编辑器中编辑项目
- **THEN** 切换到其他编辑器再切换回来时
- **AND** 动画帧序列编辑器恢复之前的编辑状态
- **AND** 未保存的更改仍然存在

### 资源库集成
#### Requirement: 素材资源库共享
系统必须支持在不同编辑器之间共享素材资源库。

#### Scenario: 在动画编辑器中导入资源
- **WHEN** 用户在动画帧序列编辑器中导入PNG纹理
- **THEN** 该纹理同时添加到素材资源库
- **AND** 在其他编辑器中也可以使用该纹理
- **AND** 资源库显示纹理的预览和元数据

#### Scenario: 使用资源库中的素材
- **WHEN** 用户在动画帧序列编辑器中打开资源库
- **THEN** 显示所有可用的纹理和动画资源
- **AND** 用户可以将资源拖拽到编辑器中使用
- **AND** 资源使用记录更新，便于查找常用资源

## ADDED Requirements

### 动画帧序列编辑器集成
#### Requirement: 骨骼动画编辑器深度集成
系统必须将骨骼动画编辑器深度集成到游戏素材创作工作流中。

#### Scenario: 从角色立绘创建动画
- **WHEN** 用户在角色立绘编辑器中创建了角色图像
- **THEN** 可以在动画帧序列编辑器中导入该图像作为骨骼纹理
- **AND** 基于该图像创建骨骼动画
- **AND** 动画可以导出并在游戏中使用

#### Scenario: 动画素材发布
- **WHEN** 用户在动画帧序列编辑器中完成动画制作
- **THEN** 可以点击"发布"按钮将动画添加到游戏素材库
- **AND** 动画素材包含预览图、元数据和动画文件
- **AND** 其他游戏开发模块可以使用该动画素材

### 工作流优化
#### Requirement: 一体化创作体验
系统必须提供从静态图像到动态动画的一体化创作体验。

#### Scenario: 完整角色创作流程
- **WHEN** 用户需要创建游戏角色
- **THEN** 可以在同一个页面中：
  1. 创建角色立绘（静态图像）
  2. 创建Q版战棋模型
  3. 创建骨骼动画（动态）
  4. 创建技能图标和状态图标
- **AND** 所有素材自动关联和组织
- **AND** 可以批量导出为游戏资源包
