## 一、阿里云通义（推荐）

### 基本信息
- **官方网址**: https://dashscope.aliyun.com
- **文档**: https://help.aliyun.com/zh_CN/dashscope
- **支持语言**: Java, Python, Go, NodeJS
- **国内节点**: ✅ 低延迟（北京、杭州）

### 优势
✅ **中文优化最好** - 针对中文的语义理解和生成
✅ **API 文档完整** - 示例代码丰富，社区支持好
✅ **商业支持稳定** - 企业级 SLA 保证
✅ **成本可控** - 按量付费，可预估
✅ **多模态能力** - 文本生成 + 图像生成 + 理解都有
✅ **集成生态好** - 与阿里云其他服务无缝集成

### 劣势
⚠️ **价格相对较高** - 比百度贵 20-30%
⚠️ **文案隐喻** - 某些特殊场景理解略欠火候

### 文字生成模型
| 模型 | 能力 | 参数量 | 推荐场景 |
|------|------|--------|---------|
| `qwen-plus` | 通用 | 13B | 日常任务，成本低 |
| `qwen-max` | 强语义 | 72B | 复杂推理，立绘文案 |
| `qwen-max-longcontext` | 长文本 | 32K tokens | 长文档理解 |

### 图像生成模型
| 模型 | 能力 | 推荐场景 |
|------|------|---------|
| `wanx-v1` | 标准 | **立绘首选** |
| `wanx-sketch-to-image` | 图生图 | 图像改写 |
| `flux-pro` | 高保真 | 高质量输出 |

### 定价（2024年）
```
文本生成 (万个 token):
- qwen-plus: ¥1-3
- qwen-max: ¥5-10

图像生成 (每张):
- wanx-v1: ¥0.5-1（大小类别不同）
- 高分辨率: ¥1-2
```

### 集成代码示例

```java
// Maven 依赖
<dependency>
    <groupId>com.aliyun</groupId>
    <artifactId>alibabacloud-dashscope</artifactId>
    <version>1.0.0</version>
</dependency>

// 配置
aliyun.dashscope.api.key=sk-xxxxxxxx

// 使用
AliyunTongYiService tongyi = new AliyunTongYiService();
List<String> images = tongyi.generateImage(
    "日系二次元少女",
    1024,
    1024,
    4,
    12345L
);
```

### Java 8 支持
✅ 完全支持 Java 8
✅ 不需要特殊适配

## 六、推荐选型方案

### 方案 A: 单一模型（推荐）

**选择: 阿里云通义**

- 最稳定、最可靠
- 中文支持最好
- 社区资源最丰富
- 商业支持最完善

```java
// 配置
ai.portrait.default-provider=aliyun
aliyun.dashscope.api.key=sk-xxxxxxxx
```

### 方案 B: 双模型备份（成熟后）

**主模型: 阿里云通义**
**备选: 百度文心**

优势:
- 提高系统可用性（主模型宕机时自动切换）
- 可根据需求选择不同模型
- 成本优化（低峰期使用百度）

```java
// 数据库设计已支持 provider 字段
// 前端可选择使用哪个模型

ai.portrait.default-provider=aliyun
ai.portrait.fallback-provider=baidu
```

### 方案 C: 成本优化（成熟后）

主要用百度文心，定点使用阿里云（高质量需求）

优势:
- 成本最低
- 足够满足大多数需求

劣势:
- 需要微调提示词
- 某些场景效果欠佳

## 九、常见问题解答

### Q1: 如何选择模型大小？
**A**:
- `qwen-plus`: 14B，推荐日常使用
- `qwen-max`: 72B，复杂推理，立绘文案
- 对于立绘生成，`qwen-max` 的提示词理解更好

### Q2: 成本如何控制？
**A**:
- 设置每日 quota 限制
- 实现请求排队机制
- 使用缓存避免重复请求
- 监控异常请求

### Q3: 如何处理模型宕机？
**A**:
- 实现自动重试机制（3 次）
- 集成备选模型（百度）
- 用户界面显示友好错误提示
- 记录所有失败请求便于分析

### Q4: 如何监控生成质量？
**A**:
- 记录所有生成的参数和结果
- 用户反馈评分系统
- 定期分析提示词有效性
- A/B 测试不同模型的表现

### Q5: 可以离线部署国内模型吗？
**A**:
- 部分开源模型可以（如 Qwen, Baichuan）
- 但本方案推荐使用 API，理由：
  - 无需本地 GPU
  - 维护成本低
  - 自动获得模型更新
