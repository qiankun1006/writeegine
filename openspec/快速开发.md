# 🚀 前端开发指南

## 快速开发工作流

### 方式 1：**最快的开发方式**（推荐日常开发）

这是改完代码立即看效果的方式，**无需重启 Spring Boot**。

#### 第一次设置（一次性）
```bash
# 1. 进入前端目录
cd src/main/resources/static/ai-portrait-generator

# 2. 安装依赖
npm install
```

#### 每次开发时（后续每次开发都用这个流程）
```bash
# 终端 1：启动 Vite 开发服务器（自动监听文件变化）
cd src/main/resources/static/ai-portrait-generator
npm run dev

# 终端 2：保持 Spring Boot 运行（不用重启）
# 只需确保 Spring Boot 已经启动即可
```

#### 访问开发页面
在浏览器中访问：
```
http://localhost:5173/static/asset-editors/character-portrait.html?dev=true
```

**工作原理**：
- Vite 开发服务器运行在 5173 端口
- 自动检测代码变化并热更新
- 页面会自动刷新显示最新的改动
- 不需要 `npm run build` 也不需要重启 Spring Boot

---

### 方式 2：**生产部署方式**（打包发布）

当你完成开发，需要部署到生产环境时使用。

```bash
# 进入前端目录
cd src/main/resources/static/ai-portrait-generator

# 构建生产版本
npm run build

# 构建完成后，Spring Boot 会自动加载最新的生产文件
# 无需重启 Spring Boot（如果已启用文件监听）或重启 Spring Boot
```

访问页面（正常的生产 URL）：
```
http://localhost:8083/create-game/asset/character-portrait
```

## 详细说明

### 🔥 开发模式自动检测机制

文件 `character-portrait.html` 中已经内置了开发模式检测：

```javascript
const isDevelopment = new URLSearchParams(window.location.search).get('dev') === 'true' ||
                     location.hostname === 'localhost' ||
                     location.hostname === '127.0.0.1';
```

- 如果访问 URL 中有 `?dev=true` → 开发模式
- 如果本机访问（localhost / 127.0.0.1）→ 开发模式

**开发模式工作流**：
1. 优先尝试从 Vite Dev Server（http://localhost:5173/）加载应用
2. 如果 Vite Dev Server 未运行，自动回退到生产构建
3. 支持热模块替换（HMR），代码改动会实时反映在浏览器中

## 常见问题排解

### Q1: 改了代码但页面没有更新？
**A**: 检查以下步骤：
1. ✅ Vite 开发服务器是否在运行？（终端应该显示 `VITE ... ready in XXXms`）
2. ✅ 浏览器访问的 URL 是否是 `http://localhost:5173/...`？
3. ✅ 浏览器是否禁用了缓存？（按 F12 打开开发者工具，勾选 "Disable cache"）
4. ✅ 检查浏览器控制台是否有错误信息

### Q2: 如何从开发模式切换到生产模式？
**A**:
- 生产模式：`npm run build` → 刷新页面（或重启 Spring Boot）
- 开发模式：运行 `npm run dev`，然后访问 `http://localhost:5173/` 开头的 URL

### Q3: 为什么有时候需要清除浏览器缓存？
**A**: Vite 开发模式使用 HMR（Hot Module Replacement），但某些情况下可能需要硬刷新：
```
Ctrl + Shift + R （Windows/Linux）
Cmd + Shift + R （Mac）
```

### Q4: 可以同时开发多个 Vue 项目吗？
**A**: 可以。Vite 端口配置在 `vite.config.ts` 中：
```javascript
server: {
  port: 5173,
}
```
如果 5173 被占用，可以改为其他端口，然后相应修改 HTML 中的代理 URL。
