# 各引擎适配层实现分析

## 1. 适配层架构概述

DragonBones 采用统一的适配层架构设计，通过工厂模式和抽象接口实现与不同游戏引擎的集成。核心架构包括：

### 1.1 统一接口设计

所有引擎适配层都继承自 `BaseFactory`，实现统一的接口：

```46:78:src/main/resources/templates/不是本项目功能，只是用来参考和学习的开源代码，不要改动/DragonBones/src/dragonBones/factory/BaseFactory.ts
export abstract class BaseFactory {
    protected static _objectParser: ObjectDataParser = null as any;
    protected static _binaryParser: BinaryDataParser = null as any;

    protected readonly _dragonBonesDataMap: Map<DragonBonesData> = {};
    protected readonly _textureAtlasDataMap: Map<Array<TextureAtlasData>> = {};

    public constructor(dataParser: DataParser | null = null) {
        if (BaseFactory._objectParser === null) {
            BaseFactory._objectParser = new ObjectDataParser();
        }

        if (BaseFactory._binaryParser === null) {
            BaseFactory._binaryParser = new BinaryDataParser();
        }

        this._dataParser = dataParser !== null ? dataParser : BaseFactory._objectParser;
    }
```

### 1.2 抽象方法定义

适配层需要实现的关键抽象方法：

- `_buildTextureAtlasData()`：构建纹理集数据
- `_buildArmature()`：构建骨架
- `_buildSlot()`：构建插槽
- `_isSupportMesh()`：是否支持网格

    public constructor(dataParser: DataParser | null = null, useSharedTicker = true) {

export class CocosFactory extends BaseFactory {
    private static _dragonBonesInstance: DragonBones = null as any;
    private static _factory: CocosFactory | null = null;

    public constructor(dataParser: DataParser | null = null) {
        super(dataParser);

        if (!CC_EDITOR) { // Is playing.
            if (this._node === null) {
                const nodeName = "DragonBones Node";
                this._node = cc.find(nodeName);
                if (this._node === null) {
                    this._node = new cc.Node(nodeName);
                    cc.game.addPersistRootNode(this._node);
                }
            }

            if (!this._node.getComponent(ClockHandler)) {
                this._node.addComponent(ClockHandler);
            }

            const eventManager = this._node.getComponent(CocosArmatureComponent) || this._node.addComponent(CocosArmatureComponent);
            if (CocosFactory._dragonBonesInstance === null) {
                CocosFactory._dragonBonesInstance = new DragonBones(eventManager);
                DragonBones.yDown = false;
            }
        }
        else {
            if (CocosFactory._dragonBonesInstance === null) {
                CocosFactory._dragonBonesInstance = new DragonBones(null as any);
                DragonBones.yDown = false;
            }
        }

        this._dragonBones = CocosFactory._dragonBonesInstance;
    }

## 总结

DragonBones 的引擎适配层设计体现了良好的架构原则：

1. **统一接口**：所有适配层提供一致的 API
2. **灵活扩展**：易于添加新的引擎支持
3. **性能优化**：针对各引擎特性进行优化
4. **兼容性**：支持不同版本的引擎

这种设计使得 DragonBones 能够在多个游戏引擎中无缝运行，为开发者提供了统一的工作流程。
