# 前端开发调试指南

本文档详细介绍AI立绘生成器前端项目的热重载范围和调试方法。

## 1. 热重载开发环境

### 1.1 启动开发服务器

```bash
# 进入前端项目目录
cd src/main/resources/static/ai-portrait-generator

# 启动Vite开发服务器（热重载调试用）
npm run dev

# vue重新加载
npm run build
```

### 1.2 访问开发地址

**主要开发地址**: `http://localhost:5173/`

**或带参数的原地址**: `http://localhost:8083/create-game/asset/character-portrait?dev=true`

## 2. 热重载支持范围

### 2.1 支持热重载的文件类型

#### Vue组件文件 (`.vue`)
```vue
<!-- src/components/CoreParamsPanel.vue -->
<template>
  <!-- HTML模板修改会立即生效 -->
  <div class="core-params">
    <input v-model="prompt" placeholder="输入提示词" />
  </div>
</template>

<script setup lang="ts">
// JavaScript/TypeScript逻辑修改会立即生效
import { ref } from 'vue'
const prompt = ref('')
</script>

<style scoped>
/* CSS样式修改会立即生效 */
.core-params {
  padding: 16px;
}
</style>
```

#### JavaScript/TypeScript文件 (`.js`, `.ts`)
```typescript
// src/stores/portraitStore.ts
import { defineStore } from 'pinia'

export const usePortraitStore = defineStore('portrait', {
  state: () => ({
    prompt: '',
    negativePrompt: '',
    // 状态修改会立即生效
  })
})
```

#### 样式文件 (`.css`, `.scss`)
```scss
// src/styles/theme.scss
$primary-color: #6C5CE7;
$secondary-color: #A29BFE;

// 样式修改会立即生效
```

#### 静态资源
- 图片文件 (`.png`, `.jpg`, `.svg`)
- 字体文件 (`.woff`, `.ttf`)

## 6. 实际调试示例

### 6.1 调试CoreParamsPanel组件

```vue
<!-- src/components/CoreParamsPanel.vue -->
<script setup lang="ts">
import { ref, watch } from 'vue';

const prompt = ref('');
const negativePrompt = ref('');

// 添加调试watch
watch(prompt, (newValue) => {
  console.log('提示词变化:', newValue);
});

// 在浏览器console中可以直接访问
(window as any).coreParams = { prompt, negativePrompt };
</script>
```

在浏览器console中可以：
```javascript
// 查看当前值
console.log(coreParams.prompt.value);

// 修改值进行测试
coreParams.prompt.value = '新的提示词';
```

### 6.3 调试状态管理

```typescript
// src/stores/portraitStore.ts
import { defineStore } from 'pinia';
import { ref, computed } from 'vue';

export const usePortraitStore = defineStore('portrait', () => {
  const prompt = ref('');
  const isGenerating = ref(false);

  // 添加调试日志
  const generationStatus = computed(() => {
    console.log('生成状态变化:', {
      isGenerating: isGenerating.value,
      prompt: prompt.value
    });
    return isGenerating.value ? '生成中' : '待生成';
  });

  return {
    prompt,
    isGenerating,
    generationStatus
  };
});
```

## 8. 调试最佳实践

### 8.1 使用source map
确保在开发环境中启用source map，这样可以看到原始代码而不是编译后的代码。

### 8.2 合理命名
给组件、变量、函数起有意义的名字，便于调试时识别。

### 8.3 模块化调试
将复杂功能拆分成小组件，分别调试。

### 8.4 状态快照
在关键节点保存状态快照，便于对比分析。

### 8.5 错误边界
使用try-catch捕获异常，提供有意义的错误信息。

```typescript
try {
  // 可能出错的代码
  await generatePortrait();
} catch (error) {
  console.error('生成图片失败:', error);
  // 提供用户友好的错误信息
  showError('生成失败，请检查网络连接或稍后重试');
}
```

## 9. 常见问题解决

### 9.1 热重载不生效
- 检查Vite开发服务器是否正常运行
- 确认访问的是开发地址
- 检查文件是否在支持的热重载范围内

### 9.2 Vue DevTools不显示
- 确认Vue DevTools已正确安装
- 刷新页面重新加载DevTools
- 检查是否在开发模式下运行

### 9.3 API请求问题
- 检查Network面板中的请求状态
- 确认后端服务正常运行
- 检查CORS配置是否正确

通过这些调试方法和最佳实践，你可以高效地开发和维护Vue应用，快速定位和解决问题。
