【问题描述】
================================================================================

用户反馈：导入本地图片后，屏幕上没有显示导入的图片
症状：
  - 菜单点击正常
  - 文件选择对话框正常
  - 选择图片后无反应
  - 编辑器 Canvas 上看不到图片
  - 浏览器控制台没有错误

================================================================================
【根本原因】
================================================================================

编辑器采用分离式架构：
  ✓ Canvas 元素 - 直接 DOM 元素，用于屏幕显示
  ✓ Document 系统 - 包含多个 Layer（图层）的数据模型

原始代码的错误：
  ✗ 图片直接绘制到主 Canvas
  ✗ 没有添加到文档的图层系统
  ✗ 调用 render() 时被清空并覆盖

原始渲染流程：
  1. 直接在 Canvas 上绘制图片 ← 临时的
  2. 调用 editor.render()
  3. render() 清空 Canvas
  4. render() 从文档重新合成内容
  5. 图片因不在文档中而消失

================================================================================
【解决方案】
================================================================================

新的正确实现：
  ✓ 创建 Layer 对象
  ✓ 在图层的 Canvas 上绘制图片
  ✓ 将图层添加到文档
  ✓ render() 正确合成图层

新的渲染流程：
  1. 创建新的 Layer 对象
  2. 在图层 Canvas 上绘制图片
  3. 添加到 Document.layers
  4. 调用 editor.render()
  5. render() 合成所有图层
  6. 图片正确显示
